# bosSD 草案

**状态**：草案，未实现  
**目标 BOS 版本**：未来（≥ 1.3）  
**目的**：为 BOS 引入虚拟块设备抽象，支持外部存储设备的挂载、读写、弹出。

## 设计目标

- 允许用户将外部文件（`.bosSD`）视为独立存储设备
- 支持两种加载模式：
  - **硬件加载**：将整个设备映像载入一块特殊空间（快速，但数量有限）
  - **逻辑加载**：按需从主机文件系统读取数据（慢速，节省内存）
- 与现有 `FS` 库共用一套文件操作接口
- 支持多设备同时挂载
- 支持热插拔（挂载 / 卸载）

## 核心概念

| 概念 | 说明 | 类比 |
|------|------|------|
| `bosSD` 文件 | 一个普通文件（扩展名 `.bosSD`），内部组织为“块 + 元数据” | 虚拟磁盘镜像 |
| 硬件加载 | 将整个 `.bosSD` 文件读入内存，作为块设备使用 | RAM Disk |
| 逻辑加载 | 不预读，直接在主机文件系统上读写（每次操作访问一次文件） | 直接 I/O |
| 挂载点 | BOS 中的一个路径（如 `/sd`），访问该路径即访问设备 | Unix 挂载点 |
| 块 | 设备的最小读写单位（例如 512 字节或一个簇） | 硬盘扇区 |

## 设备操作命令（草案）

### 挂载设备

```bm
BM::SD(MOUNT "/path/to/data.bosSD", "hw")   ; 硬件加载
BM::SD(MOUNT "/path/to/data.bosSD", "sw")   ; 逻辑加载
```
成功时返回设备 ID（如 0），失败返回错误码。

卸载设备
```bm
BM::SD(UNMOUNT 0)   ; 卸载 ID 为 0 的设备
```
读写设备
不直接提供 SD(READ) / SD(WRITE)，而是通过文件系统接口访问挂载点：

```bm
BM::FS(READ "/sd/myfile.txt")
BM::FS(WRITE "/sd/newfile.bm", "content")
```
底层由 FS 库根据路径前缀（/sd）路由到对应的 bosSD 设备。

设备信息
```bm
BM::SD(INFO 0);
```
返回设备 0 的信息（容量、已用空间、加载模式等）
文件系统视图
bosSD 设备挂载后，在 BOS 的全局文件系统命名空间中表现为一个独立子树：

```text
 ...
└── sd/          ← bosSD 设备挂载点
    ├── game.bm
    ├── save.dat
    └── ...
```
对普通 BM 脚本而言，访问 /sd/... 与访问 /sys/... 没有区别。FS 库根据路径前缀自动选择后端（主 FS 或 bosSD 设备）。

弹出设备
```bm
BM::SD(EJECT 0);
```
弹出 ID 为 0 的设备
弹出操作必须完成：
将设备内部所有脏数据（已修改但未写回）写入对应的 .bosSD 文件
释放设备占用的内存或文件句柄
使挂载点（如 /sd）失效
弹出之后，.bosSD 文件处于“完整、可被其它程序使用”的状态。用户在弹出前可以安全地复制、移动或备份该文件。
